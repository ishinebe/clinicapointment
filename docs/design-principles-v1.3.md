# アポ帳生成システム 設計原則 Ver1.3

Templateをデザインマスターとして扱う方針へ更新。

## 目的

現在の紙運用を維持したまま、Excelによるアポ帳作成を自動化し、毎月の作成時間と人的ミスを削減する。

## 基本方針

1. 紙運用を前提とする。
2. Templateをデザインマスターとして扱う。
3. レイアウトと設定を分離する。
4. 通常設定と例外設定を分離する。
5. 列ごとの担当者を設定可能にする。
6. 時間列のみマクロで改善する。
7. 現場の運用を変えない。
8. 印刷品質を最優先する。
9. 拡張性を確保する。

## シート構成

| シート | 役割 | 編集者 |
|---|---|---|
| Template | 1日分のデザインマスター。列幅、行高、セル結合、罫線、フォント、余白、印刷設定を保持する。 | 管理者のみ |
| Settings | 年月、担当者マスター、列ごとの担当者選択、通常診療時間、勤務パターン等を保持する。 | 管理者 |
| Exceptions | 会議、早退、休診、臨時変更などの例外を保持する。 | 受付・管理者 |
| Output | 印刷用アポ帳。Templateをコピーし、日付・時間・担当者・網掛け等を反映して自動生成する。 | 編集しない |

## Templateシートの設計原則

- Templateは1日分のアポ帳デザインのみを保持する。
- Dr列、予備枠、DH列の列幅・配置・セル結合・罫線はTemplateを正とする。
- 時間列の表示内容はTemplateに依存せず、マクロが `9:00`, `9:15`, `9:30`, `9:45` の形式で描画する。
- Templateの列幅や行高を変更すれば、Outputの見た目にも反映される構造とする。
- Templateには患者情報や月別情報を入れず、常に再利用可能なデザインマスターとして保つ。

## マクロが上書きする対象

| 対象 | 処理内容 | 備考 |
|---|---|---|
| 日付・曜日 | 作成年月に基づいて自動入力する。 | 例：2027年1月16日（土） |
| 担当者名 | Settingsの列設定に基づいて表示する。 | 将来的に担当者ID参照へ拡張する。 |
| 時間表示 | `9:00`, `9:15`, `9:30`, `9:45` の形式で描画する。 | Templateの時間表示は使用しない。 |
| 時間外網掛け | 診療終了時刻以降に薄い網掛けを適用する。 | ボールペン記入可能な濃さにする。 |
| 休診表示 | 木曜・日曜などの休診日を表示する。 | 完全休診は濃い網掛けまたは休診表示。 |

## 処理フロー

1. Settingsから作成年月を読み込む。
2. Templateの1日分デザインをOutputへコピーする。
3. コピーしたブロックの日付・曜日を置換する。
4. Settingsに基づき担当者名を置換する。
5. 時間列をマクロで再描画する。
6. 曜日別診療時間を判定する。
7. 診療終了後、休診日、例外予定の網掛けを反映する。
8. 1か月分を繰り返し生成し、印刷範囲を設定する。

## 優先順位

1. 完全休診・祝日・臨時休診
2. Exceptionsシートの個別例外
3. 担当者ごとの勤務パターン
4. 医院全体の診療時間マスター
5. Templateシートの基本レイアウト
