# Phase 5: 例外処理・時短勤務 設計メモ

## 目的

通常の曜日別設定では表現できない、学会休み、臨時休診、特定スタッフの休み、特定スタッフの時短勤務を扱う。

受付スタッフが迷わず操作できるように、自由入力ではなくプルダウンを基本とする。

---

## 基本方針

### 1. 通常設定・当月設定・例外設定を分ける

通常の曜日別勤務は `Settings` に置く。

その月だけ全体的に変更する設定も `Settings` に置く。

特定の日付だけに起こる例外は `Exceptions` に置く。

```text
Settings   = 毎週繰り返す基本パターン + 当月全体の上書き
Exceptions = 特定の日付だけに起こる例外
```

### 2. 空欄は通常通りとする

入力が必要なのは、通常と違う場合だけ。

毎日・毎週の出勤を入力させない。

### 3. プルダウン優先

受付担当者が変わっても操作しやすいよう、以下はプルダウン化する。

- 例外種別
- 対象
- 時刻
- 担当者

---

## 入力量を増やさないための階層

同じ月の大部分に共通する変更を `Exceptions` に毎日入力させない。

以下の階層で扱う。

```text
1. 通常設定
2. 当月全体の上書き
3. 特定日の例外
```

### 当月共通終了時刻

`当月共通終了時刻` は、医院全体の診療終了時刻をその月だけ一括で早める設定である。

これはスタッフ個別の時短勤務ではない。

対象は全体であり、Dr1、Dr2、予備、DH1、DH2の全列に適用する。

候補：

```text
Settings!B16:F16: 当月共通終了時刻
```

入力例：

```text
B16:F16 = 17:00
```

この場合、その月の診療日は原則として17:00以降を全体網掛けする。

木曜・日曜などの休診日は、従来通り休診を優先する。

---

## スタッフ個別の当月時短

「Dr2は1月中だけ16:00あがり」のようなケースは、当月共通終了時刻では扱わない。

この場合は、スタッフ個別の当月時短設定として扱う。

候補：

```text
Settings!B18:F18: 当月スタッフ別終了時刻
```

列対応は、担当者見出しと同じにする。

```text
B18 = Dr1の当月終了時刻
C18 = Dr2の当月終了時刻
D18 = 予備枠の当月終了時刻
E18 = DH1の当月終了時刻
F18 = DH2の当月終了時刻
```

入力例：

```text
C18 = 16:00
```

この場合、Dr2列のみ、その月の診療日は16:00以降を網掛けする。

空欄の場合は通常通り勤務とする。

### 曜日別の当月上書き

「その月だけ水曜は17:00、土曜は16:30」のような医院全体の曜日別上書きも将来的に検討する。

候補：

```text
A20:A26 = 月〜日
B20:B26 = 当月だけの医院全体終了時刻
```

ただし、まずは以下を優先する。

```text
1. 当月共通終了時刻
2. 当月スタッフ別終了時刻
3. 特定日の例外
```

## Exceptions シート案

```text
A列: 日付
B列: 種別
C列: 対象
D列: 開始時刻
E列: 終了時刻
F列: メモ
```

### 種別プルダウン

```text
休診
時短診療
担当休
担当時短
メモ
```

### 対象プルダウン

```text
全体
Dr1
Dr2
予備
DH1
DH2
```

または、Settingsの担当者マスターから直接担当者名を選ぶ方式も将来的に検討する。

---

## 種別ごとの挙動

### 休診

医院全体が休み。

例：学会休み、年末年始、臨時休診。

処理内容：

- その日のブロック全体を網掛けする。
- 見出し付近に `休診` と表示する。

入力例：

```text
2027/1/15 | 休診 | 全体 |   |   | 学会
```

---

### 時短診療

医院全体の終了時刻が通常より早い。

処理内容：

- 終了時刻以降を全体に網掛けする。
- 通常の曜日別診療時間より優先する。
- 当月共通終了時刻よりも、特定日の `Exceptions` を優先する。

入力例：

```text
2027/1/20 | 時短診療 | 全体 |   | 17:00 | 院内研修
```

---

### 担当休

特定の担当者だけ休み。

処理内容：

- 対象列だけ終日網掛けする。
- 担当者名の横に `休` を追記する。

入力例：

```text
2027/1/22 | 担当休 | DH1 |   |   | 有休
```

---

### 担当時短

特定の担当者だけ途中で勤務終了する。

処理内容：

- 対象列だけ、終了時刻以降を網掛けする。
- 担当者名の横に `16:00まで` のように表示する。

入力例：

```text
2027/1/23 | 担当時短 | Dr2 |   | 16:00 | 早退
```

---

## 処理優先順位

同じ日に複数の設定がある場合、以下の順で優先する。

1. Exceptions の `休診`
2. Exceptions の `時短診療`
3. Exceptions の `担当休`
4. Exceptions の `担当時短`
5. Settings の曜日別担当者休み
6. Settings の当月スタッフ別終了時刻
7. Settings の当月共通終了時刻
8. 通常の曜日別診療時間

---

## 次の実装ステップ

### Phase 5-A

`Settings` に当月共通終了時刻の入力欄とプルダウンを追加する。

### Phase 5-B

`Settings` に当月スタッフ別終了時刻の入力欄とプルダウンを追加する。

### Phase 5-C

`Exceptions` シートを作成し、見出しとプルダウンを設定するマクロを追加する。

### Phase 5-D

`休診` と `時短診療` を反映する。

### Phase 5-E

`担当休` と `担当時短` を反映する。

### Phase 5-F

祝日・年末年始などの定型休診を扱うか検討する。
