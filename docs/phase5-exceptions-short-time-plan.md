# Phase 5: 例外処理・時短勤務 設計メモ

## 目的

通常の曜日別設定では表現できない、学会休み、臨時休診、特定スタッフの休み、特定スタッフの時短勤務を扱う。

受付スタッフが迷わず操作できるように、自由入力ではなくプルダウンを基本とする。

---

## 基本方針

### 1. 通常設定と例外設定を分ける

通常の曜日別勤務は `Settings` に置く。

例外的な予定は `Exceptions` に置く。

```text
Settings   = 毎週繰り返す基本パターン
Exceptions = 特定の日付だけに起こる例外
```

### 2. 空欄は通常通りとする

入力が必要なのは例外がある日だけ。

毎日・毎週の出勤を入力させない。

### 3. プルダウン優先

受付担当者が変わっても操作しやすいよう、以下はプルダウン化する。

- 例外種別
- 対象
- 時刻
- 担当者

---

## Exceptions シート案

```text
A列: 日付
B列: 種別
C列: 対象
D列: 開始時刻
E列: 終了時刻
F列: メモ
```

### 種別プルダウン

```text
休診
時短診療
担当休
担当時短
メモ
```

### 対象プルダウン

```text
全体
Dr1
Dr2
予備
DH1
DH2
```

または、Settingsの担当者マスターから直接担当者名を選ぶ方式も将来的に検討する。

---

## 種別ごとの挙動

### 休診

医院全体が休み。

例：学会休み、年末年始、臨時休診。

処理内容：

- その日のブロック全体を網掛けする。
- 見出し付近に `休診` と表示する。

入力例：

```text
2027/1/15 | 休診 | 全体 |   |   | 学会
```

---

### 時短診療

医院全体の終了時刻が通常より早い。

処理内容：

- 終了時刻以降を全体に網掛けする。
- 通常の曜日別診療時間より優先する。

入力例：

```text
2027/1/20 | 時短診療 | 全体 |   | 17:00 | 院内研修
```

---

### 担当休

特定の担当者だけ休み。

処理内容：

- 対象列だけ終日網掛けする。
- 担当者名の横に `休` を追記する。

入力例：

```text
2027/1/22 | 担当休 | DH1 |   |   | 有休
```

---

### 担当時短

特定の担当者だけ途中で勤務終了する。

処理内容：

- 対象列だけ、終了時刻以降を網掛けする。
- 担当者名の横に `17:00まで` のように表示する。

入力例：

```text
2027/1/23 | 担当時短 | DH2 |   | 17:00 | 早退
```

---

## 処理優先順位

同じ日に複数の設定がある場合、以下の順で優先する。

1. Exceptions の `休診`
2. Exceptions の `時短診療`
3. Exceptions の `担当休`
4. Exceptions の `担当時短`
5. Settings の曜日別担当者休み
6. 通常の曜日別診療時間
